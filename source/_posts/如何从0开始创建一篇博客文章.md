##### 本文记录了我在Win10系统上进行 Hexo + GitHub Pages 的博客搭建完成后，总结出的日常写作与更新流程，适合和我一样刚完成博客搭建、容易忘步骤的初学者。

------



## 一、刚开机的“起手式”，让电脑进入一个**可以写博客、可以部署**的状态

### 1.打开终端cmd

win + R 组合键

### 2.进入博客工程目录

```
cd /d D:\Blog\my-blog
```

### 3.确认当前状态干净

```
git status
```

理想情况下看到的是：

```
On branch main
nothing to commit, working tree clean
```

说明：

- 没有上次没提交的改动
- 现在是一个“干净的起点”



## 二、写一篇新文章 / 修改一篇旧文章

### 1.创建文章文件（若是修改旧文章，则跳过此步骤）

```
hexo new "今天的博客标题"
```

这条命令会：

- 自动在 `source/_posts/` 下生成一个 `.md`
- 自动写好时间、标题（Front-matter）

（**不要**自己手动在 _posts 里新建文件）

### 2.打开对应.md文件，并写内容

```
---
title: 今天的博客标题
date: 2026-01-07 23:10:00
---

这里开始写正文（Markdown）
```



## 三、更新前预览

在 **上线之前**，自己先看一遍效果。

### 本地预览

在cmd界面键入：

```
hexo s
```

再用浏览器打开：

```
http://localhost:4000
```

如果预览后发现有问题：

- **不要 push**
- 先在本地改到满意为止

停止预览：

```
Ctrl + C
```



## 四、更新到线上

标准三连

```
git add -A
git commit -m "post: 对应的博客标题"
git push
```

- `git add -A`：告诉 Git「我要提交这些改动」
- `git commit`：打一个“版本快照”
- `git push`：把快照推到 GitHub（触发自动部署）



## 五、确认博客更新

- `git push` 成功后，
- 打开GitHub所在库的**Actions**，等待对应workflow的黄色等待标志转为**绿勾**（数秒 ～ 数分钟）
- 浏览器刷新：`https://用户名.github.io`



## 六、总结：更新博客的“完整流程”

```
1. 开机，进入cmd
2. cd /d D:\Blog\my-blog
3. hexo new "标题"（若要创建新文章）
4. 写 Markdown
5. hexo s  （本地预览）
6. Ctrl + C
7. git add -A
8. git commit -m "post: 对应标题"
9. git push
10.等待GitHub的Actions中更新workflow完成
11. 在浏览器刷新 https://用户名.github.io
```



## 七、认识个人博客本地文件夹中各个文件

### 1.`_config.yml`

位置：`D:\Blog\my-blog\_config.yml`

记载整个博客的**身份信息**和**运行方式**。

#### ✅ 应该在这里写什么？

✅ 博客名 / 作者
✅ 语言 / 时区
✅ 网站地址
✅ 全局功能开关（评论、搜索、部署方式等）

#### ❌ 不该在这里写什么？

❌ 菜单
❌ 侧边栏
❌ 主题颜色
❌ 页面布局



### 2.`_config.landscape.yml`

位置：`D:\Blog\my-blog\_config.landscape.yml`

是Landscape主题的配置文件，记载**UI 层面的所有东西**。

#### ✅ 应该在这里写什么？

✅ 菜单（menu）
✅ 首页显示什么
✅ 文章摘要样式
✅ 主题特有选项

#### ❌ 不该在这里写什么？

❌ title / author / url
❌ timezone



### 3.`文章标题.md`

位置：`D:\Blog\my-blog\source\_posts`

记载**已写的文章**。

```

---
title: 我的第一篇真正博客
date: 2026-01-07
---

正文内容

```



### 4.`index.md`

位置：`source/about/index.md`

记载**页面内容**。



### 5.`themes/landscape/`

位置：`themes/landscape`

记载**主题的源码**。

包含：

- HTML 模板
- CSS
- JS
- 主题逻辑



### 6.`pages.yml`

位置：`.github/workflows/pages.yml`

**作用**是git push 后，自动生成并发布网站。
